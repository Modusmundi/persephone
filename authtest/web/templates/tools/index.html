Token Manipulation Tools

Tools for security testing - modify and re-sign authentication tokens

Security Testing Only

These tools are designed for authorized security testing and penetration testing engagements. Manipulated tokens are clearly labeled and should never be used maliciously.

JWT Token Manipulation

Modify and re-sign JWT tokens

Test how applications handle manipulated JWT tokens including:

  • Modify claims (sub, aud, iss, roles)
  • Extend expiration time
  • Strip signature (alg=none attack)
  • Algorithm confusion (RS256 to HS256)
  • Re-sign with custom key
Open JWT Tool

SAML Assertion Manipulation

Modify and re-sign SAML assertions

Test how applications handle manipulated SAML assertions including:

  • Modify NameID (user identity)
  • Change attributes and roles
  • Extend validity conditions
  • Strip XML signatures
  • Modify issuer and audience
Open SAML Tool

Common Security Test Cases

Signature Validation

  • Does the app reject tokens with stripped signatures?
  • Does it validate signatures against the correct key?
  • Is it vulnerable to algorithm confusion attacks?

Claim Validation

  • Does the app validate issuer and audience?
  • Are expiration times properly enforced?
  • Can roles/permissions be escalated?

Identity Impersonation

  • Can user identity be changed in assertions?
  • Does the app rely solely on token contents?
  • Are session bindings validated?

Authorization Bypass

  • Can admin roles be added to tokens?
  • Are custom claims properly validated?
  • Can audience restrictions be bypassed?
