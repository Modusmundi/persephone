Available Flows

Authorization Code Flow

The standard OAuth2/OIDC flow for web applications. Redirects to the IdP for authentication, then exchanges the authorization code for tokens.

Best for: Server-side web applications

Authorization Code + PKCE

Enhanced security for public clients using Proof Key for Code Exchange. Available as an option in the Authorization Code flow.

Best for: Single-page apps (SPAs), mobile/native applications

Client Credentials Flow

Machine-to-machine authentication without user interaction. The client authenticates using its client ID and secret to obtain an access token.

Best for: Server-to-server API access, background services, microservices

Implicit Flow Legacy

NOT RECOMMENDED. Returns tokens directly in the URL fragment without a code exchange step. Tokens are exposed in browser history, referrer headers, and logs.

Best for: Testing legacy implementations only. Use Authorization Code + PKCE instead.

Device Code Flow

The Device Authorization Grant for devices with limited input capabilities. The device displays a code and URL, and the user authorizes on a separate device with a browser.

Best for: Smart TVs, CLI tools, IoT devices, game consoles
