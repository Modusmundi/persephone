How IdP-Initiated SSO Works

In IdP-Initiated SSO, the authentication flow starts at the Identity Provider (IdP) rather than at the Service Provider (SP). The IdP sends an unsolicited SAML Response to the SP without a prior AuthnRequest.

Flow Steps:

  1. User logs into the IdP portal
  2. User clicks on the application link in the IdP
  3. IdP generates a SAML Response (without InResponseTo)
  4. IdP POSTs the SAML Response to the SP's ACS URL
  5. SP validates the assertion and authenticates the user
